# 笔记宪法 📜

## 核心原则
1. **格式统一**：所有笔记必须使用 Markdown 格式（`.md` 后缀）。
2. **分类存储**：按知识领域或主题分文件夹存放（如 `/AI/`、`/Programming/`）。

## 标题规范
- 标题需简洁明确，能直接概括笔记内容（示例：`## 梯度下降算法原理`）。
- 避免模糊标题（如 `## 笔记1`）。

## 内容要求
1. **模块化**：每条笔记只阐述一个核心知识点，避免长篇大论。
2. **图文对应**：
   - 图片存放目录需与笔记所在目录 **同名+`_media`后缀**（示例：笔记 `/AI/深度学习.md` → 图片 `/AI_media/梯度下降.png`）。
   - 引用路径示例：`![梯度下降](../AI_media/梯度下降.png)`（使用相对路径向上回溯）。

## 导航与链接

1. **内部链接**：鼓励在笔记中使用相对路径链接到其他相关笔记（示例：`[相关主题](./subfolder/note.md)`）。
2. **锚点支持**：可通过`#标题`进行页内跳转；跨页锚点需确保标题ID正确。
3. **目录结构**：长篇笔记应在开头添加目录（使用`[TOC]`或手动链接）。
4. **链接维护**：更新文件路径时需同步更新所有相关链接。

## 智能体效率

1. **精准分析**：避免对所有笔记进行泛化总结，仅针对当前任务所需的文档进行查看和分析。
2. **Token节约**：在满足需求的前提下，优先使用摘要、索引或关键词搜索，减少不必要的全文读取。
3. **按需加载**：处理多文件时，优先加载与任务直接相关的文件，避免一次性加载全部内容。

## 协作与维护
1. **修改记录**：任何增删改需在 Git commit 中说明原因。
2. **历史保护**：禁止直接删除历史笔记，过期内容标记为 `## [存档]标题`。

## 未来扩展
- **向量化准备**：保持内容结构化，便于未来导入向量库（如分段落、关键词标粗）。
- **双轨优化**：兼顾人类可读性（清晰排版）和机器可读性（标记关键元数据）。
